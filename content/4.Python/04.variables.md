## 1. **What Are Variables?**

- Variables are containers for storing data values in Python.
- They act as named references to data stored in memory.
- Python is dynamically typed, so no explicit type declaration is needed.

**Example:**

```bash
x = 10  # Integer variable
name = "Alice"  # String variable
price = 19.99  # Float variable
```

---

## 2. **Creating Variables**

- A variable is created when you assign a value to it using the `=` operator.
- Syntax: `variable_name = value`
- No need to declare the variable type; Python infers it from the value.

**Example:**

```bash
age = 25  # Creates an integer variable
greeting = "Hello"  # Creates a string variable
is_student = True  # Creates a boolean variable
```

**Key Points:**

- Variables are created the moment they are assigned a value.
- You can reassign a different value or type to the same variable.

```bash
x = 5  # x is an integer
x = "Python"  # x is now a string
```

---

## 3. **Variable Names**

- Variable names must follow specific rules and conventions for clarity and functionality.

**Rules:**

- Must start with a letter (a-z, A-Z) or underscore (`_`).
- Can contain letters, numbers (0-9), and underscores.
- Cannot start with a number.
- Case-sensitive (`age`, `Age`, and `AGE` are different variables).
- Cannot use Python reserved keywords (e.g., `if`, `for`, `while`).

**Conventions (PEP 8):**

- Use descriptive names (e.g., `student_name` instead of `sn`).
- Use lowercase with underscores for multi-word names (snake_case).
- Avoid single-letter names except for counters (e.g., `i` in loops).
- Constants should be in uppercase (e.g., `MAX_VALUE = 100`).

**Examples:**

```bash
# Valid variable names
user_age = 30
_total = 100
name2 = "Bob"

# Invalid variable names
2name = "Invalid"  # Starts with a number (SyntaxError)
my-var = 5  # Contains hyphen (SyntaxError)
for = "loop"  # Reserved keyword (SyntaxError)
```

## 4. **Assigning Multiple Variables**

- Python allows assigning values to multiple variables in a single line.

**Method 1: Assigning the Same Value**

```sh
x = y = z = 10
print(x, y, z)  # Output: 10 10 10
```

**Method 2: Assigning Multiple Values (Unpacking)**

- Use commas to assign multiple values to multiple variables.

```sh
a, b, c = 1, 2, 3
print(a, b, c)  # Output: 1 2 3
```

**Unpacking from Lists/Tuples:**

```bash
numbers = [10, 20, 30]
x, y, z = numbers
print(x, y, z)  # Output: 10 20 30
```

**Note:**

- The number of variables must match the number of values, or a `ValueError` occurs.

```sh
a, b = 1, 2, 3  # ValueError: too many values to unpack
```

## 5. **Outputting Variables**

- Use the `print()` function to display variable values.
- Combine variables with text using:
  - **Comma-separated values**: `print("Text", variable)`
  - **f-strings** (Python 3.6+): `print(f"Text {variable}")`
  - **String concatenation**: `print("Text " + str(variable))`

**Examples:**

```bash
name = "Alice"
age = 25

# Using commas
print("Name:", name, "Age:", age)  # Output: Name: Alice Age: 25

# Using f-strings
print(f"{name} is {age} years old.")  # Output: Alice is 25 years old.

# Using concatenation (requires type conversion for non-strings)
print("Age: " + str(age))  # Output: Age: 25
```

**Note:**

- Use `str()` to convert non-string types to strings for concatenation.
- f-strings are the most readable and recommended for modern Python.

## 6. **Global Variables**

- Variables defined outside a function are **global** and accessible throughout the program.
- Variables defined inside a function are **local** by default and only accessible within that function.

**Example: Global Variable**

```bash
x = "global"

def my_function():
    print("Inside function:", x)  # Accesses global variable

my_function()  # Output: Inside function: global
print("Outside function:", x)  # Output: Outside function: global
```

**Modifying Global Variables Inside Functions:**

- Use the `global` keyword to modify a global variable inside a function.

```bash
x = 10

def change_global():
    global x  # Declare x as global
    x = 20  # Modifies the global variable

change_global()
print(x)  # Output: 20
```

**Local vs. Global Conflict:**

- If a variable is assigned a value inside a function without `global`, Python creates a local variable.

```bash
x = "global"

def my_function():
    x = "local"  # Creates a local variable, doesn't affect global x
    print("Inside function:", x)

my_function()  # Output: Inside function: local
print("Outside function:", x)  # Output: Outside function: global
```

**Best Practices:**

- Avoid overusing global variables to prevent unintended side effects.
- Use function parameters and return values for cleaner code.
